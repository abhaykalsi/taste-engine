## How I Structure and Present Work
*Based on Damien Correll's "Steal This Deck" + personal practice*
*Last updated: 2026-05-10*

---

## Core Premise

The slide deck is the design. A deck is not a document — it is a collaboration artifact. It wraps the things you make with context. Treat it with that weight.

**Personal position:** Idea first, structure second. The idea describes the structure — not the other way around. Don't start with a template and fill it. Start with the idea and let it find its form.

No long paragraphs of information nobody looks at. Each slide should have one thought.

---

## Story Structure

Three acts regardless of topic:

| Act | Also Known As | What It Does |
|---|---|---|
| Context | What? | Current situation |
| Vision | So what? | Where we want to go |
| Strategy | Now what? | How we get there |

If the deck doesn't do at least one of these, it should be an email.

**Kurt Vonnegut story shape:** Map the audience's emotional arc. The visual shape of the story should feel like the story itself.

---

## Presentation Types

**Informational:** Linear. Setup → Information. Flat trajectory. Updates, reports, status syncs.

**Persuasive:** Setup → Solution → Feasibility check → Revised solution (the "but") → CTA. Has a dip, a climb, a moment of doubt, then resolution. For buy-in or milestones.

**Educational:** Setup → Foundation → Demo → Increased complexity → Demo → Next steps. Ladders up in complexity, each level earning the next.

**Inspirational:** Setup (small dip) → Lesson (long rising arc) → CTA. Builds slowly to emotional peak. To motivate or shift perspective.

## Format Calibration

```
Self-study ←————————→ TED Talk
Dense, text-heavy      Sparse, image-heavy
No voiceover           You are the voiceover
```

Most decks get presented live AND referred back to. Balance accordingly. Don't go full TED Talk if people will re-read this without you.

## Draft Milestones

| Draft | Goal | How |
|---|---|---|
| 1st | Make it exist | Outline. Block big ideas. Just make a file. |
| 2nd | Make it functional | String narrative. Each section sets up a takeaway. |
| 3rd | Make it effective | Amplify storytelling. Polish design. Edit for clarity. |

**Heuristic:** 1 paragraph = 1 slide when blocking initial structure from a document.

**Strive for clarity, not simplicity.** Editing is not reduction — it is bringing more clarity. The question is never quantity, it is whether each slide earns its place.

## Common Mistakes

**Too much text per slide:** First draft never edited. Break dense slides into multiples. One dense slide covering four topics → overview naming the four + four individual slides. Use colour as wayfinding (one colour per topic consistently).

**Presenting information, not ideas:** Full data table on slide, talking over it. If you're verbally highlighting one insight from a chart, why show the whole chart? Present the idea the data supports. Isolate the relevant bar. Show only what you're going to talk about.

**Lack of hierarchy:** Everything same size, weight, importance. Use scale (larger = more important), weight (bold subheads, lighter body), opacity (reducing secondary text creates grades without new colours), layout (thirds help the eye parse).

**Poor wayfinding:** Aristotle rule — tell them what you will tell them (agenda), tell them (chapters), tell them what you just told them (next steps). At each chapter break: show where you are AND where you're going.

## Design Notes for Slides

**Typography:** Scale and weight direct the eye. Typeset carefully — hanging punctuation, no orphans. Mix weights deliberately: loud display + mid-weight subhead + lighter body.

**Colour:** Start restrained (white or black backgrounds for content-heavy slides). Use colour as punctuation — at chapter breaks, on hero slides, where you want feeling not just reading. Colour as consistent signifier across sections builds memory.

**Layout:** Play with thirds. Image + text: 50/50 or slightly smaller image to give text room. Text left, image right is reliable. 12-column grid as underlying structure.

**Templates:** Exist for consistency and confidence, not constraint. Build with flexibility baked in. Start with the 5-6 slide typologies you actually use, add variants as you discover you need them.
